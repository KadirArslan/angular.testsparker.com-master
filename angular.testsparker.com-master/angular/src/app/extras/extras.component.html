Allow out-of-scope XMLHttpRequests

Description: Check this if the target website fails to load when some requests cannot be loaded because of Scan Profile out-of-scope settings.

How To Enable?: Scan Policy Editor > YOUR_POLICY > JavaScript > Check "Allow out-of-scope XML HTTP request during simulation" checkbox

Case: Report internal ip here: then loading document.

Filter Document Events

Description: Whether to filter events that are attached to "document" by name to a constant set of mousedown, keyup etc. in order to reduce triggered event count during the simulation.

How To Enable?: Scan Policy Editor > YOUR_POLICY > JavaScript > Check "Filter document Events" checkbox

Case: Triggered newMessage event after loading page and send XHR follow report Internal IP Disclosure.

Filter Colon Events

Description: Filter events that have a colon (:) Whether to filter events that contain a colon (:) in their name. They are usually used by frameworks and would be triggered by other events. Check this option in order to reduce triggered event count during the simulation.

How To Enable?: Scan Policy Editor > YOUR_POLICY > JavaScript > Check "Filter 'colon' Events" checkbox

Case: redirect with "two:dots" event. Triggered two:dots event and send XHR follow report Internal IP Disclosure.
